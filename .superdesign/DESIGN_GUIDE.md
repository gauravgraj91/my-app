# Design Guide

## Instructions
- Build one single html page of just one screen to build a design based on users' feedback/task
- ALWAYS output design files in '.superdesign/design_iterations' folder as {design_name}_{n}.html (Where n needs to be unique like table_1.html, table_2.html, etc.) or svg file
- If iterating on existing file, naming convention: {current_file_name}_{n}.html, e.g. ui_1.html → ui_1_1.html, ui_1_2.html
- ALWAYS use tools for write/edit html files, don't just output in a message

## Styling
1. Use flowbite library as a base unless the user specifies otherwise.
2. Avoid indigo or blue colors unless specified in the user's request.
3. MUST generate responsive designs.
4. When designing component, poster or any other design that is not full app, make sure the background fits well with the actual UI color; e.g. if component is light then background should be dark, vice versa.
5. Font should always use google font, default fonts: 'JetBrains Mono', 'Fira Code', 'Source Code Pro','IBM Plex Mono','Roboto Mono','Space Mono','Geist Mono','Inter','Roboto','Open Sans','Poppins','Montserrat','Outfit','Plus Jakarta Sans','DM Sans','Geist','Oxanium','Architects Daughter','Merriweather','Playfair Display','Lora','Source Serif Pro','Libre Baskerville','Space Grotesk'
6. Include !important for CSS properties that might be overwritten by tailwind & flowbite, e.g. h1, body, etc.
7. NEVER use bootstrap style blue color unless asked — use reference themes below instead.

## Images & Icons
- Images: use placeholder from unsplash, placehold.co or known public URLs — don't make up urls
- Icons: use lucide icons — `<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>`

## Script Imports
- Tailwind: `<script src="https://cdn.tailwindcss.com"></script>` (NOT the CSS link)
- Flowbite: `<script src="https://cdn.jsdelivr.net/npm/flowbite@2.0.0/dist/flowbite.min.js"></script>`

## Design Workflow
Only follow when designing a full UI screen from scratch. Skip steps for quick iterations.

1. **Layout** — Present ASCII wireframe and confirm with user
2. **Theme** — Propose colors, fonts, spacing; confirm with user
3. **Animation** — Outline key transitions/interactions; confirm with user
4. **Generate** — Write HTML file to `.superdesign/design_iterations/`
